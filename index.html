Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Hypertext Markup Language: gives content structure
  • doctype: document type
  • lang: language
  • style: CSS styles
  • link: connects an external resource
  • meta: metadata (e.g., character set, description, keywords)
  • script: embeds client-side scripts
  • element: the entire elements from opening to closing tag
  • tag: what is inside the angle brackets
  • child elements: elements nested inside other elements
  • Semantic HTML elements: make HTML easier to read, element names describe the element's content and role
  • div: creates a page division and is not semantic
  • span: creates an inline division and is not semantic
  • p: a block of text
  • id: used to identify elements and select them
  • ul: unordered list of items
  • ol: ordered list of items
  • li: list items
  • img: contains information about images
  • src: defines the image location
  • br: creates a line break (empty space)
  • a (anchor): creates webpage links
  • href: points to the URL
  • h1, h2, h3...: heading levels
  • charset: sets character encoding
  • http-equiv: sets an HTTP header for content
  • viewport: sets default view

CSS

  • Cascading Style Sheets: give content style

Git

  • git status: checks what branches we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git is a version control system
  • GitHub: cloud-based hosting service for collaboration
  • GitHub issues: used for task management and are often formatted using Markdown language
  • default branch: called the main branch
  • feature branch: a copy of the main branch and is used to add new functionality
  • changes are pushed to the feature branch and then merged to the main branch by opening a pull request
  • git command: must precede every statement
  • checkout command: moves the working branch to a new branch
  • add: adds modifications to the current working branch in staging
  • -A: indicates that we want to add all changes
  • git commit: captures the project's currently staged changes
  • pull: receives a branch's modifications into the local environment
  • origin keyword: source of pull
  • push: push local changes to remote GitHub branch

JavaScript

  • JavaScript: handle logic and allow users to interact with a website
